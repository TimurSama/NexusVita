# 📁 Как указать Root Directory в Vercel

## Проблема
Vercel может не показывать `apps/web` в списке выбора Root Directory.

## ✅ Решение

### Вариант 1: Ввод вручную (Рекомендуется)

1. В настройках проекта Vercel найдите поле **Root Directory**
2. **Введите путь вручную** (не выбирайте из списка):
   ```
   apps/web
   ```
3. Убедитесь, что:
   - Нет пробелов в начале или конце
   - Нет слеша в конце (`apps/web` ✅, `apps/web/` ❌)
   - Используется правильный регистр (все маленькие буквы)

### Вариант 2: Если поле неактивно

Если поле Root Directory неактивно или не видно:

1. **Отключите автоматическое определение фреймворка:**
   - В настройках проекта найдите **Framework Preset**
   - Выберите **Other** или **Next.js** вручную
   - После этого поле Root Directory должно стать активным

2. **Или создайте проект заново:**
   - Удалите текущий проект в Vercel
   - Импортируйте репозиторий снова
   - На этапе настройки укажите Root Directory: `apps/web`

Если Root Directory все еще не работает, можно использовать `vercel.json`:

Файл `apps/web/vercel.json` уже создан с настройками. Vercel должен автоматически определить его.

## 🔍 Проверка структуры

Убедитесь, что в вашем репозитории есть такая структура:

```
nexus-vita/
├── apps/
│   └── web/
│       ├── package.json
│       ├── next.config.mjs
│       ├── src/
│       └── ...
├── apps/
│   └── api/
│       └── ...
└── package.json
```

## ⚠️ Важно

- Root Directory указывается **относительно корня репозитория**
- Путь должен быть: `apps/web` (не `/apps/web` и не `./apps/web`)
- После указания Root Directory сохраните настройки
- Vercel автоматически пересоберет проект

## 🐛 Если не работает

1. Проверьте, что репозиторий синхронизирован с GitHub
2. Убедитесь, что папка `apps/web` существует в репозитории
3. Попробуйте сделать новый commit и push
4. Перезапустите деплой в Vercel

---

**После указания Root Directory деплой должен работать!** 🎉
