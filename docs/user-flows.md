# Пользовательские сценарии (User Flows)

## 1. Регистрация нового пользователя

1. Пользователь заходит на главную страницу
2. Нажимает "Регистрация"
3. Заполняет форму (email, пароль, имя)
4. Выбирает роль (USER или TRAINER)
5. Система создаёт профиль и аватар тела
6. Пользователь перенаправляется на главную страницу

## 2. Настройка аватара тела

1. Пользователь заходит на главную страницу
2. Видит интерактивный аватар тела
3. Кликает на части тела для отметки целей
4. Выбирает проблемные зоны
5. Сохраняет настройки
6. Система синхронизирует с программами тренировок

## 3. Выбор и подписка на тренера

1. Пользователь переходит в раздел "Тренеры"
2. Просматривает список тренеров
3. Открывает страницу тренера
4. Изучает контент и программы
5. Нажимает "Подписаться" или "Отслеживать"
6. Получает доступ к контенту тренера

## 4. Покупка абонемента

1. Пользователь переходит в "Локации"
2. Выбирает зал/центр
3. Просматривает доступные абонементы
4. Выбирает абонемент
5. Оплачивает (валюта или токены)
6. Получает цифровую карту
7. Использует QR/NFC для прохода

## 5. Создание страницы тренера

1. Тренер регистрируется с ролью TRAINER
2. Переходит в настройки страницы
3. Выбирает тему оформления
4. Настраивает модули (лента, календарь, магазин)
5. Заполняет информацию о себе
6. Публикует страницу
7. Получает уникальный URL

## 6. Онлайн-тренировка

1. Тренер создаёт событие в календаре
2. Указывает время и тип (онлайн/офлайн)
3. Пользователи записываются на событие
4. В назначенное время получают уведомление
5. Переходят в видео-комнату
6. Проводят тренировку
7. Система записывает результат

## 7. Участие в челлендже

1. Пользователь видит активные челленджи
2. Выбирает интересующий челлендж
3. Изучает условия и награды
4. Присоединяется к челленджу
5. Выполняет задания
6. Отслеживает прогресс
7. Получает награды и токены

## 8. Использование токенов

1. Пользователь выполняет активности (тренировки, челленджи)
2. Получает токены за выполнение
3. Просматривает баланс на главной странице
4. Переходит в магазин
5. Выбирает товар/услугу
6. Оплачивает токенами
7. Получает доступ к покупке

## 9. Демо-режим

1. Пользователь заходит на страницу /demo
2. Выбирает роль для изучения
3. Выбирает конкретную функцию
4. Система активирует демо-режим
5. Пользователь изучает функционал
6. Все действия не сохраняются
7. Может переключиться на другую роль/функцию
