# Архитектура Nexus Vita

## Обзор

Nexus Vita построена как модульная экосистема с разделением на фронтенд и бэкенд.

## Технологический стек

### Фронтенд
- **Next.js 14** - React фреймворк с SSR
- **TypeScript** - типизация
- **Tailwind CSS** - стилизация
- **Framer Motion** - анимации
- **Zustand** - управление состоянием
- **Axios** - HTTP клиент

### Бэкенд
- **Node.js + Express** - сервер
- **TypeScript** - типизация
- **Prisma** - ORM для работы с БД
- **PostgreSQL** - база данных
- **JWT** - аутентификация
- **Socket.io** - WebSocket для чата

## Структура проекта

```
nexus-vita/
├── frontend/              # Next.js приложение
│   ├── src/
│   │   ├── app/          # Страницы и роутинг
│   │   ├── components/   # React компоненты
│   │   ├── contexts/     # React контексты
│   │   └── lib/          # Утилиты
│   └── public/           # Статические файлы
├── backend/              # Express API
│   ├── src/
│   │   ├── routes/       # API роуты
│   │   ├── middleware/   # Middleware
│   │   └── server.ts     # Точка входа
│   └── prisma/           # Схема БД
└── docs/                 # Документация
```

## Модель данных

Основные сущности:
- **User** - пользователи всех ролей
- **TrainerProfile** - профили тренеров
- **TrainerPage** - мини-сайты тренеров
- **Theme** - темы оформления
- **Post** - публикации
- **WorkoutPlan** - программы тренировок
- **NutritionPlan** - планы питания
- **Location** - локации (залы, центры)
- **Product** - товары и услуги
- **Order** - заказы
- **TokenTransaction** - транзакции токенов
- **Chat/Message** - чаты и сообщения
- **Event** - события (тренировки, вебинары)
- **Challenge** - челленджи

## Система ролей

1. **GUEST** - гость (только просмотр)
2. **USER** - пользователь (клиент)
3. **TRAINER** - тренер/специалист
4. **LOCATION_MANAGER** - менеджер локации
5. **ADMIN** - администратор

## Система тем

Темы хранятся в БД и применяются через CSS переменные:
- Каждая тема имеет конфигурацию (цвета, шрифты, текстуры)
- Темы могут быть системными или созданными пользователями
- Применение темы меняет `data-theme` атрибут на `<html>`

## API

Все API эндпоинты находятся под `/api/*`:
- `/api/auth` - аутентификация
- `/api/users` - пользователи
- `/api/trainers` - тренеры
- `/api/posts` - публикации
- `/api/workouts` - тренировки
- `/api/nutrition` - питание
- `/api/locations` - локации
- `/api/shop` - магазин
- `/api/chat` - чаты
- `/api/events` - события
- `/api/challenges` - челленджи
- `/api/tokens` - токены
- `/api/themes` - темы

## Безопасность

- JWT токены для аутентификации
- Хеширование паролей (bcrypt)
- Middleware для проверки прав доступа
- CORS настройки
- Helmet для защиты заголовков

## Масштабируемость

- Модульная архитектура компонентов
- Разделение на микросервисы (при необходимости)
- Кэширование на уровне API
- Оптимизация запросов к БД через Prisma
