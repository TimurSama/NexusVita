# ⚠️ КРИТИЧЕСКИ ВАЖНО: Root Directory в Vercel

## Проблема

Vercel показывает ошибку: "The specified Root Directory 'Nexus Vita' does not exist"

## Причина

В GitHub репозитории файлы находятся **в корне репозитория**, а не в подпапке "Nexus Vita".

Структура в GitHub:
```
NexusVita/ (корень репозитория)
  package.json
  app/
  components/
  ...
```

## Решение

### В настройках Vercel:

1. Откройте **Settings** → **General**
2. Найдите раздел **Root Directory**
3. **УДАЛИТЕ** значение "Nexus Vita" (оставьте поле **ПУСТЫМ**)
4. Нажмите **Save**

### Почему пустое?

- Git репозиторий находится в папке "Nexus Vita" локально
- Но когда файлы коммитятся, они попадают в **корень** репозитория GitHub
- Vercel клонирует репозиторий и видит файлы в корне
- Поэтому Root Directory должен быть **пустым**

## После исправления

1. Перейдите на вкладку **Deployments**
2. Нажмите **Redeploy** на последнем деплое
3. Или сделайте новый коммит - деплой запустится автоматически

---

**Если это не помогло:**
Проверьте, что в GitHub репозитории файлы действительно в корне, а не в подпапке.
